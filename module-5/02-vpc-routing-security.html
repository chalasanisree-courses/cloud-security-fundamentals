2 The VPC and its subnets

A VPC owns a block of addresses — a CIDR range like 10.0.0.0/16 (65,536 addresses). Subnets carve that block into smaller streets. Click a piece of the example:

AWS reserves 5 IPs per subnet
A /24 subnet has 256 addresses but only 251 usable — AWS keeps the first four (network, router, DNS, future) and the last (broadcast) in every subnet.

3 Public or private? The route table decides

Subnets are physically identical. A subnet is “public” only because its route table sends internet-bound traffic to the Internet Gateway. Change the route table and the subnet’s nature flips:

Same subnet, three route tables

Public-capable ≠ publicly reachable
A public route table only makes a subnet able to face the internet. A resource is actually reachable only if it also has a public IP and the NACL and Security Group allow the traffic. Routing and security are separate jobs.

4 NACL vs Security Group

Two filters guard your traffic. The NACL is the cop at the street entrance; the Security Group is the gate on each driveway. The difference that trips everyone up is stateless vs stateful — follow a reply and watch:

5 Three gates, in order: defense in depth

A packet must clear three checkpoints to reach a host: the route table (is there a path?), the NACL (does the subnet allow it?), the Security Group (does the host allow it?). Pick a journey:

Can the packet get there?

Remember

A VPC owns a CIDR block (/16 = 65,536 addresses); subnets carve it up, and AWS reserves 5 IPs per subnet. A subnet is public only because its route table points 0.0.0.0/0 at an Internet Gateway — but public-capable isn’t publicly reachable without a public IP and permitting rules. Private subnets reach out through a NAT Gateway. Two filters enforce security: the NACL guards the subnet (stateless, ordered, allow + deny) and the Security Group guards the instance (stateful, allow-only). Together — route table, NACL, Security Group — they are defense in depth.
