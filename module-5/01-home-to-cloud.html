1 Start at home

Your home network is one subnet of devices on private IP addresses, all reaching the internet through a single box. That box quietly does five different jobs — click each:

The home router — one box, five jobs

The home assumption
A home network treats all inbound traffic from the internet as unsafe and blocks it by default, while letting your outbound connections (and their replies) through. Hold onto that assumption — the enterprise has to break it.

2 NAT: many devices, one address

Private IPs like 192.168.1.10 can’t travel the public internet. Network Address Translation rewrites them to the router’s one public IP on the way out, and reverses it on the way back. Watch a packet make the round trip:

Network Address Translation

NAT is not a firewall
NAT only maps addresses. Unsolicited inbound traffic gets dropped simply because there’s no entry in the translation table to send it to — a side effect, not a security policy. Don’t mistake NAT for protection.

3 Scale up: home → enterprise → cloud

The same principles hold as networks grow — only the implementation changes. The single home box explodes into specialized hardware, then dissolves into software in the cloud. Step through the three:

Same principles, three scales

Why the DMZ exists
Enterprises must let strangers in — customers hitting a website, partners calling an API — without exposing internal databases. The fix is the DMZ: a buffer segment that hosts the internet-facing systems and is isolated from the internal network, so a compromised web server can’t reach straight into your data. It limits the blast radius.

4 The VPC is your enterprise network, in software

An AWS VPC is the cloud version of the enterprise perimeter — logically isolated, even though it runs on shared hardware. Every piece has a familiar counterpart. Click across:

Enterprise concept → AWS equivalent

An analogy, not a definition
“DMZ = public subnet” is a useful mental hook, but not literal. A real DMZ is a deliberately screened buffer for internet-facing services; an AWS subnet is merely “public” because its route table points at an Internet Gateway. The next note makes that routing precise.
Remember

Networking principles are the same from home to cloud — only the implementation changes. A home router collapses five jobs (routing, NAT, firewall, gateway, Wi-Fi) into one box and blocks inbound by default. NAT lets private IPs share one public address but is not a firewall. An enterprise explodes that box into specialized hardware and adds segmentation — notably a DMZ to accept inbound safely. An AWS VPC is that enterprise network in software: enterprise net → VPC, edge router → Internet Gateway, NAT device → NAT Gateway, DMZ → public subnet, internal net → private subnet.
