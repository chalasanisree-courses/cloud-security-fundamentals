A Note on Ports

Module 1 Foundations Quick reference
Learning Objectives
  • Define a port and explain how it differs from an IP address
  • Explain why a single machine needs ports to run many services at once
  • Recognize the common ports and the services that listen on them
  • Understand that a port is a logical number, not physical hardware

1 What is a port?

A port is a number the operating system uses to decide which application should receive incoming network traffic. In the Networking lecture you saw an IP address get a packet to the right machine; the port gets it to the right program on that machine.

  • An IP address identifies the computer.
  • A port number identifies the application on that computer.

You always need both.

The one-line version
IP address = which machine.   Port number = which program on that machine.

2 Why ports exist

A single computer often runs many network services at the same time — for example a web server, a remote-login service, and an email service, all on one machine. Without ports, the operating system would have no way to know which program should receive a given piece of incoming data.

Ports solve this by acting as logical doorways into different applications. A packet arrives at the device's IP address; the OS reads the destination port number and hands the data to whichever application is listening on that port. In fact, the packet you watched travel in the Networking note carries two port numbers alongside its IP and MAC addresses — a source port and a destination port, in the TCP header. That's why the Wireshark capture showed traffic going to :443.

Think of it like an apartment building: the IP address gets the packet to the building (203.0.113.10), and the port to the right apartment (:443).

How a packet finds the right application — pick a destination port

3 Common ports

A handful of port numbers are reserved by convention for well-known services. These are worth memorizing — they show up constantly in firewall rules, security groups, and packet captures.

PortServicePurpose
21FTPFile transfer (unencrypted)
22SSHRemote login
25SMTPEmail sending
53DNSName resolution
80HTTPWeb traffic (unencrypted)
443HTTPSWeb traffic (encrypted)
Why this matters for security
Every open port is a way in. Firewalls — and in AWS, security groups — work by allowing or blocking specific ports, so closing unused ports directly shrinks the attack surface. We configure exactly this in later modules.
Remember

A port is not hardware — not a cable, and not the USB or Ethernet jack on your laptop, despite sharing the name. A port is a logical number used by the operating system to route incoming traffic to the right application. IP gets you to the machine; the port gets you to the program.
