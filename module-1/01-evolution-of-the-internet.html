Evolution of the Internet

Module 1 Foundations Lecture 1
Learning Objectives
  • Explain why pre‑internet computing left organizations as isolated information islands
  • Trace the six eras of internet evolution and the problem each one solved
  • Identify the foundational protocols — TCP/IP, then HTTP, SMTP, FTP — and when they emerged
  • Connect each era's expansion to the steadily growing security surface this course must defend

1 Where we are

This is the first lecture of the course, and it deliberately starts before security. Everything we will defend — networks, hosts, data, identities — rides on the internet, so we begin by understanding how the internet came to be and why it looks the way it does today. We'll trace that story across six eras, and the key idea is that each era solved a problem the previous one created — while quietly enlarging the attack surface.

Pre‑internet: islands of computing
Computers were mainframes — massive machines filling entire rooms, owned by large institutions like universities and governments. Users connected through dumb terminals. Mainframes at different institutions were wired together only in proprietary ways, so there was no standard way for them to talk to each other. The result was information silos — islands of data that couldn't reach one another.

3 The protocols that built the Web

Connecting computers only matters if they can exchange information. In the 1990s the World Wide Web brought together the protocols that made exchange possible:

ProtocolStands forWhat it does
HTTPHyperText Transfer ProtocolA client requests a web page (e.g., a course syllabus) from a web server, which returns it.
SMTPSimple Mail Transfer ProtocolSending email across the internet.
FTPFile Transfer ProtocolUploading and transferring files.

Once information was flowing, it had to be findable — which is why search engines mattered, and why Google rose to dominance. The 1990s "browser wars" between Microsoft and Netscape were a fight over the client end, because whoever owned the browser shaped the user's entire internet experience. Google ultimately controlled both ends: the client (Chrome's leading share) and the server (search).

4 The AI stack

The 2020s are the age of AI. To see where AI actually sits, compare the traditional computing stack with the new AI stack — the layers line up, but the middle changes.

Traditional stack vs AI stack

The Platform layer of the traditional stack is replaced by the Model & Orchestration layer — the new heart of the stack, where the large language models live.

  • Application layer — the user-facing AI apps (Copilot, ChatGPT, chatbots).
  • Model & Orchestration layer — where the large language models (LLMs) such as GPT-5 live. This is the layer that makes the stack “AI.”
  • Infrastructure layer — the hardware (GPUs and cloud) that runs the models.
The layer to remember
In the AI stack, the Model (Model & Orchestration) layer is where the LLMs — like GPT-5 — actually live. Applications sit on top of it; infrastructure sits beneath it.

5 The through-line — and why it matters for security

Read top to bottom, the eras are a single argument: each layer was added onto the one before it.

◈ Each era added surface areaMore connectivity, more to defend ▶

Standard connections (TCP/IP) made the internet possible → standard information exchange (the Web) made it useful → mobility (networks + devices) made it everywhere → the cloud made applications location‑independent → IoT connected everything → AI now sits on top, improving every experience.

Every one of those steps added more endpoints, more data in motion, and more things an attacker can reach. That expanding surface is exactly why the rest of this course exists.

Remember
The internet is not one invention — it's a stack of standards, each layered on the one before. Pre‑internet = isolated islands; the whole story is the progressive removal of barriers between machines. Convenience and exposure grew together.
