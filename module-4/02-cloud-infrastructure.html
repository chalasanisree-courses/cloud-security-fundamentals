2 The cloud is a landlord

One cloud hosts thousands of tenants. Each gets a private, walled-off space — on AWS, a VPC (Virtual Private Cloud). Click a tenant to see what they can and can't reach:

Multi-tenant isolation

Why the VPC matters for security
The VPC is the cloud's isolation boundary — logical separation between you and every other tenant on the same shared hardware. It's also the network layer of defense in depth, and the whole subject of the next module.

Security cuts both ways
A reviewed, hardened template deploys identically every time — but one bad line (an open security group, a public bucket) is reproduced everywhere. Infrastructure as Code is misconfiguration as code too, so templates get security-reviewed like any other code.

Remember

Traditional infrastructure means siloed teams (storage, systems, network) whose coordination is the bottleneck. The cloud is a multi-tenant landlord giving each tenant an isolated VPC — the network boundary covered next. Old resources map onto services: compute → EC2, networking → VPC subnets/route tables (+ Internet Gateway), storage → S3. You provision by writing code — Infrastructure as Code, reconciled by Terraform — and ship through a continuous DevOps loop.
