Module 7 Encryption Access control

1 Two questions about your data

Securing data at rest comes down to two questions, and each has its own control:

Access control vs encryption

Access control
Who is allowed to reach the data? Keeps the wrong people out in the first place.
Encryption
What if someone gets the data anyway? Makes it unreadable without the key.
Two layers, one idea
Access control prevents access; encryption limits the damage if access control fails. They’re defense in depth for the data itself — you want both.

2 Encryption: in motion and at rest

Data is exposed in two places — while it travels to S3, and while it sits on disk. Each gets its own encryption. Click through:

Where encryption happens

Encrypted by default since 2023
You no longer have to turn it on: since January 2023, S3 applies server-side encryption to every new object automatically. Data is never unencrypted at rest by default. The choice that remains is key management — who holds the key decides who can decrypt: S3 can manage keys for you, or you can control them yourself with KMS or your own keys.

3 Access control: who gets in

Three tools decide who can read or write an object. Bucket policies do the everyday work; ACLs are legacy; and Block Public Access is the guardrail that catches mistakes. Run a request through them:

Will the request reach the object?

The safe defaults (since April 2023)
New buckets now ship locked down: Block Public Access is ON and ACLs are disabled (access is managed by bucket policies and IAM). Treat bucket policies as the primary tool and ACLs as a legacy mechanism you usually leave off. Most famous S3 leaks were buckets accidentally left open before these defaults existed.
Remember

Data-layer security has two jobs: access control (keep the wrong people out) and encryption (make stolen data useless) — access control prevents access, encryption limits the damage if it fails. S3 encrypts data in motion with TLS and, since 2023, at rest by default; the real choice is key management. Access is decided by bucket policies (the primary tool), legacy ACLs, and the Block Public Access guardrail. The rule: deny by default, allow only if a policy grants it and nothing explicitly denies it — an explicit deny always wins, and Block Public Access blocks anonymous public access no matter what the policies say.
