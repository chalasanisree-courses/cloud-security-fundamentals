1 A key needs a vault

Encryption only helps if the key stays secret. But if malware compromises the OS, anything sitting in normal memory — including your master key — is up for grabs. The fix: keep the key in an isolated vault. Toggle the attack:

Key in memory vs key in a vault

The vault does two jobs
The isolated environment doesn’t just store the key — it also performs the encryption and decryption inside itself. So the key never leaves the vault in plaintext, even to do its work. Think of a password manager on your phone: malware can infect the OS, yet the master secret stays sealed away.

3 Envelope encryption

There’s a catch: the vault is built to guard keys, not to crunch gigabytes — it won’t directly encrypt a 1 GB file (AWS KMS won’t directly encrypt more than ~4 KB). So we use two keys, like sealing a letter inside an envelope:

Master key vs working key

KEK — the master key
Never leaves the vault. Its only job is to encrypt (“wrap”) other keys — it never touches your actual data.
DEK — the working key
Generated by the vault, used by the CPU. It encrypts the actual data out in fast main memory, then the plaintext copy is destroyed.

4 The workflow

Watch a data key’s round trip. Notice the master key (KEK) never leaves the vault — only the data key travels, and only ever wrapped when it’s stored. Flip between the two directions:

Encrypt · decrypt

What gets stored next to your data
Only the encrypted DEK — the data key locked by the KEK. Without the vault (and permission to use the KEK), that wrapped key is useless, so an attacker who copies your encrypted file and the wrapped key still can’t read anything.
Remember

A key is only safe if it lives in an isolated vault that both stores it and does the crypto inside — so the key never leaves, even if the OS is owned. Those vaults are the TPM (a motherboard chip), the TEE / Secure Enclave (an enclave in the processor), and the HSM (a cloud-scale appliance). Because a vault guards keys rather than bulk data, we use envelope encryption: a KEK (master key, never leaves the vault) wraps a DEK (working key) that the CPU uses to encrypt the data. Only the wrapped DEK is stored next to the ciphertext; decrypting means sending it back to the vault to be unwrapped.
